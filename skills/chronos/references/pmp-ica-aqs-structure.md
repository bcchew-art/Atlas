# ICA AQS PMP Structure Reference

Real-world ATT PMP delivered for ICA Appointment & Queue System project.
Use this for structural reference — the depth and organisation of a real ATT PMP deliverable.
Do NOT copy content — this is format/structure reference only.

## Top-Level Structure

1. REVISION HISTORY
2. INTRODUCTION
   - Purpose
   - Definitions, Acronyms and Abbreviations
   - References
3. Project Objective
4. Project Description and Benefits
   - Project Background
   - Business Results
   - Project Cost
5. Project Scope
   - Customer Base
   - Major Deliverables
   - Interfaces
   - Assumptions
6. PROJECT ORGANISATION
   - Project Structure
   - Roles and Responsibilities
     - Product Owner and Client User Representatives
     - Project Management Team (PMT)
     - Vendor Project Team
     - Subject Matter Experts (SMEs)
7. PROJECT CONTROL MECHANISMS
   - Project Monitoring and Tracking
   - Project Escalation
   - Project Change Control
8. RISK MANAGEMENT

## Key Observations

- **Project context is substantial** — ICA AQS PMP includes full project background, business results, and cost breakdown. This is a living project document, not a tender submission. For tender PMP, we simplify: skip cost breakdown, focus on approach.
- **Org structure is client-vendor split** — clearly distinguishes client roles (Product Owner, User Reps) from ATT roles (PM, Tech Lead, Dev Team). This makes responsibilities unambiguous.
- **Project Control Mechanisms are named explicitly** — "Project Monitoring and Tracking", "Project Escalation", "Project Change Control" are each a subsection, not just bullet points. Evaluators respond well to this level of specificity.
- **Risk Management is a standalone top-level section** — not buried under governance.

## For Tender Submissions (CAG Section 9 equivalent)

In a tender PMP, the document is prospective — it describes *how you will manage the project*, not the status of an ongoing one. Key differences:
- Use future tense: "ATT will establish...", "The Project Manager will report..."
- Populate the risk register with likely risks, not actual encountered ones
- Gantt is proposed, not actuals
- CVs are submitted separately in Schedule 6 format
